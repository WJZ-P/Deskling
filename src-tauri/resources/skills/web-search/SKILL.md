# 网页搜索（web-search）

用 DuckDuckGo 搜索互联网，返回排名靠前的若干条结果。

## 用法

用 `run_command` 运行本技能目录下的 `search.cjs`（把 load_skill 返回的技能目录绝对路径拼在前面），查询词作为参数：

```
node "<技能目录>/search.cjs" "查询关键词"
```

可选第二个参数指定返回条数（默认 5，最多 20）：

```
node "<技能目录>/search.cjs" "查询关键词" 8
```

脚本把结果打印到标准输出，每条含序号、标题、链接、摘要。拿到后用其中的链接和摘要来回答用户；需要某个网页的完整正文时，可再用 `run_command` 跑 `curl` 之类抓取该链接。

## 依赖与注意

- 需要机器上装有 Node.js（脚本只用内置的 http/https 模块，Node 14 以上即可，无第三方依赖）。若报「'node' 不是命令」，说明未安装 Node——把这个情况直接告诉用户，不要臆造搜索结果。
- 若报「被 DuckDuckGo 限流或要求人机验证」，说明请求被反爬拦了（不是没结果）——告诉用户稍后重试或换网络，别把它当成「搜不到」。
- 结果来自抓取 DuckDuckGo 网页版，偶尔可能为空或被限流；为空时可换个措辞重试一次。
- 联网默认走系统代理：Deskling 会把 Windows 系统代理（若已启用）自动注入命令环境，脚本据 `HTTPS_PROXY`/`HTTP_PROXY` 走代理（很多机器访问外网需要，无需手动设置）。若报「网络不通或需要代理」，多半是本机没开系统代理或代理不通——把错误如实转告用户，让其检查代理。
- 绝不要编造搜索结果或链接：只根据脚本真实输出作答，搜不到就说搜不到。
